Laporan Analisis Keselamatan Sistem

Dokumen ini memperincikan protokol keselamatan, mekanisme pengesahan, dan integriti data yang diimplementasikan dalam ekosistem MyPEKEMA.

Ringkasan Eksekutif

Sistem MyPEKEMA telah dibangunkan dengan mengutamakan keselamatan data dan akses terkawal. Melalui gabungan teknologi OAuth 2.0 untuk pengesahan identiti dan prinsip Senarai Putih (Allowlist) yang ketat, sistem ini meminimumkan risiko pencerobohan akaun secara signifikan.

Infrastruktur keselamatan ini juga mematuhi amalan terbaik industri dengan penggunaan Prepared Statements untuk menangkis serangan SQL Injection dan kawalan akses berasaskan peranan (RBAC) untuk memastikan pemisahan tugas yang jelas antara Pentadbir dan Pengguna.

1. Pengesahan Hibrid (Hybrid Auth)

  • Google OAuth 2.0 SSO: Sistem tidak menyimpan kata laluan tempatan. Semua log masuk disahkan melalui infrastruktur Google.
  • Domain Restriction: Akses dihadkan secara keras kepada domain organisasi @customs.gov.my sahaja.
  • 2-Factor Authentication (2FA): Mewarisi tetapan keselamatan akaun Google pengguna (Google Authenticator / SMS).

2. Kawalan Akses (RBAC)

  • Sistem Whitelist: Hanya e-mel yang didaftarkan dalam pangkalan data (users_auth) dibenarkan masuk, walaupun pengesahan Google berjaya.
  • Admin vs User: Halaman sensitif seperti Pengurusan Pengguna dilindungi kod peringkat pelayan yang menyemak peranan sesi.

3. Integriti Pangkalan Data

  • Prepared Statements: Semua input pengguna diproses menggunakan parameter terikat (bound parameters) untuk membasmi sepenuhnya ancaman SQL Injection.
  • Sanitasi Input: Data dibersihkan (trimmed, type-casted) sebelum disimpan untuk mengekalkan kebersihan data.

4. Audit & Penyamaran

  • Jejak Audit: Medan last_login dikemaskini secara automatik pada setiap sesi untuk memantau keaktifan akaun.
  • Error Suppression: Paparan ralat sistem dimatikan pada persekitaran produksi untuk mengelakkan kebocoran maklumat struktur pelayan (Information Disclosure).
